Request headers

We don't have a request object because it is structured differently based on whether it is buffered or streaming, chunked or http2, etc.

Note: we do our best to keep Head object same for HTTP 1-2 and HTTPS

Fields

client

The client IP and port. If the connection is not using a standard TCP-IP connection this field will be None.

version

The HTTP protocol version.

method

The HTTP method. It is restricted to token chars.

scheme

The HTTP scheme is a sequence of characters beginning with a letter and followed by any combination of letters, digits, plus, period or hyphen.

path

The path points to a specific resource.

headers

A slice of HTTP headers.

body_kind

The body kind is either fixed, chunked or upgrade.
